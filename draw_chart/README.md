# рҹ“Ҡ Draw Chart: Three-Stage Processing Pipeline

## рҹҺҜ Overview

**Draw Chart** is a three-stage processing pipeline for analyzing federated learning experimental results:

- **Stage 01**: Data Input & Chart Generation
- **Stage 02**: Data Processing & Analysis  
- **Stage 03**: Final Results & Aggregation

Each stage is independent and can be executed in sequence.

## рҹҡҖ Quick Start

### Option 1: Full Pipeline (Recommended)

**Day 1:**
```bash
# Step 1: Prepare input
mkdir stage01/PUT-DATA-THERE/experiment01
cp -r /path/to/experiment/* stage01/PUT-DATA-THERE/experiment01/

# Step 2: Generate charts
cd draw_chart/stage01
python generator.py

# Step 3: Transfer to stage02
python move_to_stage02.py
```

**Day 2:**
```bash
# Step 4: Analyze in stage02
cd ../stage02
python generator.py
python move_to_stage03.py

# Step 5: View final results
cd ../finally_result
ls -la
# Results are ready for reports!
```

### Option 2: Manual Control

Run each stage independently:
```bash
# Stage 01 only (generate charts local to put-data)
cd stage01
python generator.py

# Later, when ready: move to stage02
python move_to_stage02.py

# Stage 02 processing (independent analysis)
cd ../stage02
python generator.py

# When satisfied: finalize to finally_result
python move_to_stage03.py
```

### Option 3: Bypass Stages

Use `raw_data/` from project root directly:
```bash
# Copy raw_data directly to stage01/PUT-DATA-THERE
cp -r ../../raw_data stage01/PUT-DATA-THERE/raw_experiment

# Continue with standard flow
cd stage01
python generator.py
python move_to_stage02.py
```

## рҹ“Ӣ What Each Stage Does

### Stage 01: Generation
**Input:** Raw experiment data  
**Action:** Generate visualization charts  
**Output:** PNG/CSV charts in `generate_charts/`  
**Next:** Auto-transfer to Stage 02

```bash
cd stage01
python generator.py        # Generate charts from PUT-DATA-THERE/
python move_to_stage02.py # Package and send to stage02
```

### Stage 02: Analysis
**Input:** Data from Stage 01  
**Action:** Analyze and compare results  
**Output:** Statistical analysis and charts  
**Next:** Auto-transfer to Stage 03

```bash
cd stage02
python generator.py        # Secondary analysis
python clearner.py        # Optional: validate data quality
python move_to_stage03.py # Finalize to finally_result
```

### Stage 03: Archive
**Input:** Processed results from Stage 02  
**Action:** Store in organized archive  
**Output:** Ready for reports and presentations  
**Next:** Use for presentations, publications, etc.

No scripts needed here - just browse and use files:
```bash
ls finally_result/*/generate_charts/*.png
```

## рҹҺҜ Use Cases

### Use Case 1: Compare Multiple Experiments
```
1. Place all experiments in stage01/PUT-DATA-THERE/
   - experiment_A/
   - experiment_B/
   - experiment_C/

2. Run stage01 generator вҶ’ creates comparison charts

3. Run move_to_stage02.py вҶ’ organizes results

4. Run stage02 generator вҶ’ produces ranking and statistics

5. Run move_to_stage03.py вҶ’ final aggregated results in finally_result
```

### Use Case 2: Batch Processing
```
Loop 1: Add data вҶ’ Stage01 вҶ’ Stage02 вҶ’ Stage03
Loop 2: Add data вҶ’ Stage01 вҶ’ Stage02 вҶ’ Stage03
Loop 3: Add data вҶ’ Stage01 вҶ’ Stage02 вҶ’ Stage03
...
```

### Use Case 3: Re-analyze Existing Results
```
1. Place results in stage02/PUT-DATA-THERE/
2. Skip stage01 entirely
3. Run stage02 generator with new parameters
4. Move to stage03
```

## рҹҡЁ Important Notes

### Data Flow
- **Data flows forward**: Stage01 вҶ’ Stage02 вҶ’ Finally_Result
- вқҢ No data flows backward
- рҹ”„ You can pause between stages and resume later

### Preservation
- Original data in `put-data/` is NOT automatically deleted
- Stage01 and Stage02 outputs are COPIED (not moved) to next stage
- Finally_Result is the final archive - treat as read-only

### Naming Convention
```
{DataDistribution}-Round-{NumRounds}-Epoch-{LocalEpochs}-{RunID}
Example: IID-Round-200-Epoch-3-20260324-2043-57
```

### Troubleshooting
```bash
# If stage02 doesn't receive data from stage01:
cd stage01
python move_to_stage02.py  # Re-run transfer

# If stage03 seems empty:
ls -R stage02/            # Check if stage02 has results
cd stage02
python move_to_stage03.py # Re-run transfer

# To clean and restart:
rm -rf stage01/generate_charts/*
rm -rf stage02/PUT-DATA-THERE/*
rm -rf stage3/PUT-DATA-THERE/*
# Then start fresh
```
